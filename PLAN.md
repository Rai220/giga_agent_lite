# GigaAgent Lite — План доработок

## 1. Поддержка работы с файлами: PDF, Excel, CSV

### 1.1 Чтение файлов

- **CSV**: парсинг через встроенный код (split по разделителям) или библиотеку (например `papaparse`)
- **Excel (.xlsx)**: чтение через `SheetJS` (`xlsx` npm-пакет), извлечение данных из листов
- **PDF**: извлечение текста через `pdf.js` (`pdfjs-dist`)

### 1.2 Инструменты агента

| Инструмент | Описание |
|-----------|----------|
| `read_csv` | Читает CSV-файл, возвращает данные как JSON-массив |
| `read_excel` | Читает Excel-файл (указанный лист), возвращает данные как JSON |
| `read_pdf` | Извлекает текст из PDF-файла |

### 1.3 Загрузка файлов

- Добавить кнопку загрузки файлов в UI (drag & drop + file input)
- Загруженные файлы хранятся в памяти (или в виртуальной FS) и доступны агенту через инструменты
- Поддержка множественной загрузки

## 2. Вывод визуального контента от агента

### 2.1 Графики

- Агент генерирует код графика (Chart.js / Plotly.js / встроенный Canvas API) внутри `execute_js`
- Результат `execute_js` может возвращать не только текст, но и изображение (base64 PNG / SVG / HTML с графиком)
- В UI: если результат содержит изображение или HTML — рендерим inline в чате

### 2.2 Изображения

- Поддержка вывода base64-изображений и blob-URL в сообщениях агента
- Результат инструмента `generate_image` (если подключён) отображается как картинка в чате

### 2.3 Таблицы (CSV)

- Если агент возвращает CSV / табличные данные — рендерим как HTML-таблицу в чате
- Возможность скачать таблицу как CSV-файл

### 2.4 Реализация

- Расширить sandbox iframe: разрешить рендеринг Canvas, возвращать `canvas.toDataURL()` как результат
- Добавить библиотеку визуализации (Chart.js) внутрь sandbox iframe
- Формат ответа инструмента: `{ type: "text" | "image" | "html" | "table", content: string }`
- UI: компонент `RichOutput` для рендеринга разных типов результатов

## 3. Проверка: генерация графиков по Excel

### Сценарий тестирования

1. Загрузить Excel-файл с данными (продажи, метрики и т.п.)
2. Попросить агента: "Построй график продаж по месяцам из этого файла"
3. Агент должен:
   - Вызвать `read_excel` — получить данные
   - Вызвать `execute_js` — сгенерировать график (Chart.js)
   - Вернуть изображение графика
4. Пользователь видит график в чате

### Что проверить

- Корректное чтение Excel с несколькими листами
- Правильное извлечение данных для осей графика
- Графики разных типов: bar, line, pie
- Отображение графика в чате (не сломана вёрстка, читаемый размер)
- Работа с большими файлами (1000+ строк)

## 4. Создание и выгрузка документов

### 4.1 Генерация документов агентом

| Формат | Библиотека | Описание |
|--------|-----------|----------|
| CSV | Встроенная генерация | Агент формирует CSV-строку |
| Excel (.xlsx) | `SheetJS` (`xlsx`) | Агент создаёт workbook с данными |
| PDF | `jsPDF` | Агент генерирует PDF-документ |
| Markdown / TXT | Встроенная генерация | Простой текст |

### 4.2 Инструменты агента

| Инструмент | Описание |
|-----------|----------|
| `create_document` | Создаёт документ (csv/xlsx/pdf/md/txt), возвращает ссылку для скачивания |

### 4.3 UI

- Результат `create_document` отображается как карточка с именем файла и кнопкой "Скачать"
- Скачивание через `Blob` + `URL.createObjectURL` + `<a download>`
- Предпросмотр: для CSV — таблица, для PDF — встроенный просмотр, для остальных — текст

## 5. Память агента

### 5.1 Концепция

Агент может запоминать информацию о пользователе и контексте между сессиями. Память сохраняется в `localStorage` и автоматически включается в system prompt.

### 5.2 Инструменты агента

| Инструмент | Описание |
|-----------|----------|
| `memory_save` | Сохранить факт/заметку в память. Параметры: `key` (тема), `content` (текст) |
| `memory_list` | Показать все сохранённые записи памяти |
| `memory_delete` | Удалить запись по ключу |

### 5.3 Как работает

1. При старте сессии — загружаем все записи памяти из `localStorage`
2. Записи включаются в system prompt в блоке:
   ```
   ## Твоя память о пользователе
   - [имя]: Пользователя зовут Алексей
   - [язык]: Предпочитает ответы на русском
   - [стек]: Работает с Python и React
   ```
3. Агент сам решает, когда сохранить что-то в память (или пользователь просит "запомни")
4. System prompt инструктирует агента использовать память

### 5.4 Хранение

- `localStorage` ключ: `agent_memory`
- Формат: `Record<string, { content: string, createdAt: string, updatedAt: string }>`
- Лимит: ~50 записей (чтобы не раздувать system prompt)

### 5.5 UI

- Раздел "Память" в настройках — список сохранённых записей
- Возможность вручную редактировать / удалять записи
- Индикатор количества записей в памяти

## TODO: Think tool и Critic tool — рассуждение и самокритика

Основано на статье Anthropic: https://www.anthropic.com/engineering/claude-think-tool

### Think tool

Инструмент для структурированного размышления. Не получает новой информации и не меняет состояние — просто добавляет мысль в лог. Позволяет агенту остановиться и подумать перед действием.

- [ ] Добавить инструмент `think`:
  ```json
  {
    "name": "think",
    "description": "Используй этот инструмент, чтобы подумать. Он не получает новой информации и не меняет состояние — просто записывает мысль в лог. Используй, когда нужно сложное рассуждение, анализ результатов, проверка плана или кэширование промежуточных выводов.",
    "parameters": {
      "type": "object",
      "properties": {
        "thought": {
          "type": "string",
          "description": "Мысль, которую нужно обдумать."
        }
      },
      "required": ["thought"]
    }
  }
  ```
- [ ] Обработчик: просто возвращает `"OK"` (или пустую строку) — результат не важен, важен сам факт рассуждения в контексте
- [ ] В UI: блок мысли показывается свёрнутым (expandable), чтобы не засорять чат
- [ ] В system prompt добавить инструкцию:
  ```
  ## Использование think tool
  Перед выполнением действий и после получения результатов инструментов используй think tool как черновик:
  - Перечисли, какие правила применимы к текущему запросу
  - Проверь, вся ли необходимая информация собрана
  - Убедись, что запланированное действие корректно
  - Проанализируй результаты инструментов на правильность
  ```

### Critic tool

Инструмент для самокритики — агент оценивает качество своей работы и находит ошибки.

- [ ] Добавить инструмент `critic`:
  ```json
  {
    "name": "critic",
    "description": "Используй этот инструмент, чтобы критически оценить свои действия или план. Запиши, что может пойти не так, какие есть слабые места, и как можно улучшить подход.",
    "parameters": {
      "type": "object",
      "properties": {
        "criticism": {
          "type": "string",
          "description": "Критический анализ текущего подхода, плана или результата."
        }
      },
      "required": ["criticism"]
    }
  }
  ```
- [ ] Обработчик: аналогично think — возвращает `"OK"`, результат не важен
- [ ] В UI: показывается свёрнутым блоком с меткой "Самокритика"
- [ ] В system prompt:
  ```
  ## Использование critic tool
  После выполнения сложных задач или перед финальным ответом используй critic tool:
  - Оцени, нет ли ошибок в твоём решении
  - Проверь крайние случаи
  - Подумай, есть ли лучший подход
  - Если нашёл проблему — исправь перед ответом пользователю
  ```
